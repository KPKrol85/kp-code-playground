# Website Scope Builder

## Overview
Website Scope Builder is a Digital Vault planning asset that helps freelancers and studios define website scope clearly before estimating and pitching work.

## Purpose of the tool
- Reduce ambiguity before pricing
- Build proposal-ready scope summaries
- Identify scope risks early
- Improve client alignment and approval flow

## Scope modules
1. Project Basics
2. Page Structure
3. Content Scope
4. Design Scope
5. Functional Features
6. E-commerce / Payments
7. SEO & Analytics
8. Technical Setup
9. Accessibility & Compliance (baseline, not certified audit)
10. Delivery & Handoff
11. Maintenance / Aftercare

## Project basics guide
Capture client/project context first: who the client is, what the website must achieve, who it serves, and when it needs to launch. Use internal notes for negotiation boundaries or technical caveats.

## Complexity scoring explanation
The builder uses weighted points:
- 1 = simple
- 2 = medium
- 3 = advanced
- 5 = high-impact/custom
- each custom page = +2

The total score estimates implementation complexity rather than price.

## Scope category interpretation
- **Starter Scope (0–20):** focused launch or simple presence.
- **Standard Website Scope (21–45):** complete business website with core requirements.
- **Advanced Website Scope (46–75):** larger scope with stronger UX/integration/SEO demands.
- **Complex / Custom (76+):** operationally complex work requiring custom estimate strategy.

## Common scope risks
- unclear content ownership
- e-commerce/legal/test flow expansion
- timeline mismatch vs selected complexity
- missing technical access credentials
- compliance/privacy requirements not supplied
- custom feature ambiguity and undefined acceptance criteria

## Sample client questions
- Who owns final copy and when will it be delivered?
- Which pages are mandatory for launch vs phase 2?
- Do you need checkout, shipping, or tax handling in phase 1?
- Who controls domain, DNS, and hosting access?
- Are legal/privacy pages supplied by your legal advisor?
- Which analytics and conversion events are business-critical?

## Sample scope report template
- Client + project basics
- Page and template list
- Content and design responsibilities
- Functional and technical requirements
- Accessibility/compliance baseline items
- Delivery/handoff and aftercare options
- Complexity score + category
- Risk warnings
- Recommended next steps
- Freelancer notes

## Practical recommendations for freelancers/studios
- Lock page list before pricing.
- Separate custom/e-commerce items from baseline web scope.
- Confirm stakeholder approvals for content, design, and launch.
- Use phased delivery when complexity is high.
- Keep written acceptance criteria for custom interactions and integrations.

## Using the generated report in proposals or agreements
Paste the plain-text report into proposals, SOWs, and agreement drafts as a "Scope Definition" section. Keep risks and next steps visible so scope assumptions are explicit. Convert large or risky items into optional add-ons, phased milestones, or separate estimates before contract signature.
