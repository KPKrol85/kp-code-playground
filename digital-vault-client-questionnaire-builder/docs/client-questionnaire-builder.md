# Client Questionnaire Builder

## Overview
Client Questionnaire Builder is a static KP_Code Digital Vault tool that helps generate tailored website discovery questionnaires.

## Purpose
Collect structured information before pricing, scoping, and production.

## Discovery workflow
1. Enter project context.
2. Choose project type, depth, and tone.
3. Select modules.
4. Generate and review questionnaire.
5. Send client-facing questions and use internal notes for planning.

## Project context guide
Capture client name, project name, industry, business model, current site status, main goal, depth, tone, and internal notes.

## Modules
1. Business & Goals
2. Audience & Positioning
3. Offer / Products / Services
4. Brand & Visual Direction
5. Content & Assets
6. Website Structure
7. Features & Functionality
8. SEO / Analytics / Marketing
9. Technical Access & Constraints
10. Timeline / Budget / Decision Process

## Project-type-specific guidance
The generator appends specific question groups for Landing page, E-commerce, Existing website redesign, Web app / dashboard, Blog / content site, Portfolio, and Personal brand website.

## Questionnaire depth
- **Quick**: fast discovery (high-priority).
- **Standard**: balanced pre-proposal set.
- **Deep Discovery**: comprehensive discovery for complex projects.

## Sample client intro
Hi [Client Name], before preparing the final proposal, I’d like to collect details about goals, content, scope, and technical setup. This keeps the project accurate and easier to price.

## Sample structure
- Header and project context
- Client intro message
- Questions grouped by selected modules
- Project-specific section
- Internal freelancer notes
- Recommended next steps

## Internal freelancer notes
Use these to protect scope quality (decision maker, content ownership, technical access, migration/SEO safeguards, and launch constraints).

## Recommended next steps
- Send questions before proposal call.
- Clarify ambiguous answers in discovery.
- Convert answers into scope, milestones, assumptions, and risks.

## Turn answers into scope/proposal
Map answers into:
- Deliverables and page/functionality list
- Responsibilities matrix (client vs freelancer)
- Timeline and milestone approvals
- Technical dependencies and risk list
- Pricing model and scope boundaries
