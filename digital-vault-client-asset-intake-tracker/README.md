# Client Asset Intake Tracker

Client Asset Intake Tracker is a standalone KP_Code Digital Vault product designed to help service providers collect, validate, and manage required client assets before production begins.

## Who it is for
- Freelancers building websites or digital products
- Web designers and developers handling client onboarding
- Small web studios and digital agencies managing multiple project handoffs

## What problem it solves
Projects are delayed when core assets, approvals, or technical access are incomplete. This tool creates one clear readiness workflow to identify missing items early and keep pre-production organized.

## What it includes
- Project intake snapshot for key client and timeline details
- Dynamic readiness dashboard (assets tracked, delivered, approved, blocked)
- Structured multi-category asset checklist
- Blockers capture area for critical risks
- Client request summary section for handoff emails and printable updates
- Intake instructions for clients
- After-intake action plan checklist
- Print-friendly styling for worksheet and PDF export workflows

## When to use it
Use this tracker during discovery-to-onboarding and before full project production starts.

## How to use it
1. Fill out the **Project Intake Snapshot** with project details.
2. Review each asset item and set **Priority** (Critical / Important / Optional).
3. Update **Status** as the client submits assets.
4. Add notes for dependencies, unresolved questions, or ownership concerns.
5. Monitor the **Readiness Dashboard** to assess production preparedness.
6. Log key issues in **Project Blockers**.
7. Draft client follow-ups in **Client Request Summary**.
8. Use the **After-Intake Action Plan** to move to production with confidence.

## Asset status explanation
- **Missing**: Asset has not been provided.
- **Requested**: Asset has been asked for and is pending delivery.
- **Received**: Asset was delivered and is awaiting quality or compliance review.
- **Reviewed**: Asset has been checked and is close to final acceptance.
- **Approved**: Asset is accepted for production use.
- **Blocked**: Asset or access cannot proceed due to an issue.

## Readiness scoring explanation
- Delivered assets include statuses: **Received, Reviewed, Approved**.
- Readiness percentage = (Delivered Assets / Total Tracked Assets) × 100.
- Labels:
  - **0–30%**: Not ready to start
  - **31–60%**: Partially ready
  - **61–85%**: Production can start carefully
  - **86–100%**: Ready for production
- Blocked assets trigger a stronger warning recommendation.

## Suggested client workflow
1. Share this worksheet at kickoff.
2. Request all critical items first (brand, copy, and technical access).
3. Track submissions weekly until blockers are resolved.
4. Send client summaries for missing assets and deadlines.
5. Approve readiness and move to production.

## Customization notes
- Add or remove asset categories based on service scope.
- Adjust priority defaults for your team process.
- Edit helper text to align with your legal, compliance, or technical standards.
- Extend action-plan items for internal QA and handoff steps.
