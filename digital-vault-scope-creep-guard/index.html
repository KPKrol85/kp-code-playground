How to use this tool

  1. Capture the request in one sentence before discussing implementation details.
  2. Set each field based on current facts, not assumptions.
  3. Use the suggested response as a draft, then personalize with project specifics.
  4. Save your final decision in project notes or your ticketing system for traceability.

Common scope creep signals

  • “This should be quick” paired with a new feature or integration request.
  • Late-stage changes that require design and development updates together.
  • Requests dependent on client assets that are not yet approved or available.
  • Post-launch asks that resemble roadmap items, not bug fixes.

Recommended workflow

  • Step 1: Assess and score request within 2–3 minutes.
  • Step 2: Send client response draft and confirm next step.
  • Step 3: If score is 8+, log a formal change request before coding.
  • Step 4: Re-baseline timeline and budget once approved.
