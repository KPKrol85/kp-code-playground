# Client Readiness Scorecard

## Overview
The Client Readiness Scorecard helps freelancers and studios evaluate whether a website client is ready for implementation. It reduces delivery risk by scoring clarity, assets, access, decision flow, budget fit, timing, and communication before kickoff.

## Categories
1. Business Clarity
2. Content Readiness
3. Brand & Visual Assets
4. Technical Access
5. Decision-Making
6. Budget & Scope Fit
7. Timeline Readiness
8. Communication Quality

Each category contains 5 readiness checks.

## Scoring scale
- **0** = not ready / missing
- **1** = partially ready
- **2** = ready

Maximum score: **80 points**.

## Interpretation guide
- **0–39% (High Risk Client):** Major blockers exist. Do not start production.
- **40–69% (Needs Preparation):** Work can proceed only after preparation tasks are complete.
- **70–84% (Almost Ready):** Mostly ready, but resolve remaining risks before build.
- **85–100% (Ready to Start):** Healthy readiness for scope, timeline, and delivery.

## Manual-use version of the assessment
Use each question with a 0/1/2 score:

### 1) Business Clarity
- Clear website goal
- Defined main offer
- Specific target audience
- Primary user action
- Unique selling points

### 2) Content Readiness
- Page copy availability
- Product/service descriptions
- Company/contact details
- Legal content awareness
- Content approver identified

### 3) Brand & Visual Assets
- Logo quality/format
- Visual direction (colors/fonts/style)
- Photo/graphic readiness
- Reference websites
- Desired brand style clarity

### 4) Technical Access
- Domain access
- Hosting access
- Business email access
- Analytics/Search Console/Tag Manager access (if needed)
- Technical ownership clarity

### 5) Decision-Making
- Final decision maker confirmed
- Stakeholder count controlled
- Feedback consolidation method
- Approval stages defined
- Response availability

### 6) Budget & Scope Fit
- Realistic budget
- Scope understanding
- Extra feature cost awareness
- Payment terms accepted
- MVP vs full system alignment

### 7) Timeline Readiness
- Realistic deadline
- Materials ready before start
- Dependency awareness
- Known blockers identified
- Revision buffer

### 8) Communication Quality
- Clear answers
- Respect for channels
- Ability to explain expectations
- Openness to recommendations
- Understanding of client role

## Report template
- Product name
- Assessment date
- Client name
- Project name
- Website type
- Final score and percentage
- Readiness status
- Category scores
- Main risks
- Recommended next steps
- Freelancer notes

## Practical recommendations for freelancers/studios
- Require a readiness score threshold before locking launch dates.
- Keep scope in writing and tie estimates to approved content/assets.
- Treat low Technical Access scores as a launch-risk red flag.
- Tie timelines to dependency milestones, not only final deadline.
- Use scorecard results to justify discovery workshops or phased delivery.
