Czym jest produkt

Biblioteka fundamentów wizualnych dla projektów frontendowych

CSS Design Token Systems to kuratorska biblioteka KP_Code Digital Vault zawierająca 200 gotowych do skopiowania systemów tokenów CSS. Każdy system opisuje konkretny kontekst UI: od SaaS i fintechu, przez edukację, zdrowie oraz panele administracyjne, po produkty gamingowe, commerce i impact tech.

Produkt nie jest generatorem ani frameworkiem. To lekki panel edukacyjny, który pomaga szybko porównać kierunki wizualne, zobaczyć je w przykładowych komponentach i przenieść wybrany fundament do własnej aplikacji.

Podstawy

Co to są design tokeny?

Design tokeny to nazwane wartości projektowe zapisane jako CSS custom properties. Zamiast wpisywać w komponentach przypadkowe kolory, odstępy i cienie, używasz zmiennych takich jak --dt-color-primary, --dt-space-4 albo --dt-radius-lg.

KoloryTła, tekst, akcenty, powierzchnie i stany interfejsu.
TypografiaRodziny fontów, rozmiary tekstu i wysokości linii.
LayoutOdstępy, promienie, szerokości kontenerów i rytm układu.
EfektyCienie, obramowania, przejścia i podstawowe wartości motion.

Praca w panelu

Jak wybrać system tokenów?

  1. Określ kontekst produktu. Zacznij od typu interfejsu: dashboard, landing page, panel operacyjny, edukacja, commerce lub narzędzie specjalistyczne.
  2. Użyj wyszukiwarki i kategorii. Filtry pomagają zawęzić 200 zestawów do kilku kierunków pasujących do problemu.
  3. Sprawdź podgląd komponentów. Każda karta pokazuje przykładowy układ, przyciski, metryki, kartę i paletę kolorów.
  4. Wybierz fundament, nie finalny layout. Tokeny mają nadać rytm, hierarchię i charakter wizualny. Komponenty aplikacji możesz później dopasować do własnego produktu.

Kopiowanie CSS

Jak skopiować i zaimportować wybrany plik?

Przycisk Kopiuj CSS na karcie zestawu kopiuje pełną zawartość odpowiadającego mu pliku z folderu tokens/. Możesz wkleić ten CSS do własnego arkusza albo zaimportować plik bezpośrednio, jeśli zachowujesz strukturę projektu. 

<link rel="stylesheet" href="./tokens/tokens-02.css" />

Po imporcie mapuj własne komponenty na zmienne --dt-*. Dzięki temu możesz zmieniać charakter UI przez podmianę jednego pliku tokenów, bez przepisywania całego systemu komponentów.

Organizacja

Jak są zbudowane pliki tokenów i panel?

Każdy plik tokens/tokens-xx.css jest samodzielnym systemem. Zawiera prymitywy kolorów, tokeny semantyczne, typografię, odstępy, promienie, cienie, wartości layoutu oraz warstwę klas demonstracyjnych w BEM.

Panel korzysta z wygenerowanych danych, aby wyświetlać karty, opisy, tagi, palety i kod CSS do kopiowania. Pliku assets/vault-data.js nie należy edytować ręcznie — dane produktu powinny pochodzić ze źródła generującego, aby panel i tokeny pozostały spójne.

Klasy podglądowe

Co oznaczają klasy preview?

Klasy z prefiksem dt- pokazują, jak tokeny zachowują się w typowych elementach UI. Możesz potraktować je jako inspirację, test wizualny albo punkt startowy do własnych komponentów.

.dt-preview
Główny kontener przykładowego widoku.
.dt-preview__title
Nagłówek demonstrujący skalę typografii i kontrast.
.dt-button / .dt-button--primary
Przyciski pokazujące akcje podstawowe i warianty powierzchni.
.dt-card
Karta do sprawdzania tła, obramowania, promieni i cienia.
.dt-metric
Mały blok danych do oceny hierarchii liczb i etykiet.
.dt-scale__swatch
Próbka koloru z palety danego systemu.

Rekomendowany proces

Pracuj od fundamentu do komponentów

  1. Wybierz 2–3 zestawy pasujące do domeny produktu.
  2. Porównaj podglądy i wybierz system o najlepszej czytelności dla Twoich treści.
  3. Skopiuj CSS lub zaimportuj wybrany plik tokenów.
  4. Podłącz komponenty aplikacji do zmiennych --dt-*.
  5. Dopracuj komponenty lokalnie, nie rozbijając spójności kolorów, odstępów i typografii.

Gotowy, aby wybrać system?

Wróć do panelu, przefiltruj bibliotekę i skopiuj CSS najlepiej dopasowany do Twojego projektu.
