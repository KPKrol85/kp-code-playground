Binary Search / Wyszukiwanie binarne

Naucz się algorytmu, który zamiast sprawdzać każdy element po kolei, w każdej iteracji odrzuca połowę możliwych odpowiedzi.

Czas: 25–35 minWymagania: tablice, pętle, warunkiZłożoność: O(log n)

Definicja

Binary Search, czyli wyszukiwanie binarne, to algorytm znajdowania elementu w posortowanej kolekcji. Porównuje szukaną wartość ze środkiem aktualnego zakresu, a następnie kontynuuje pracę tylko w lewej albo prawej połowie.

Jaki problem rozwiązuje?

Gdy dane są posortowane, nie musimy sprawdzać każdego elementu. Binary Search odpowiada na pytanie: „czy element istnieje i gdzie się znajduje?” znacznie szybciej niż Linear Search dla dużych zbiorów.

Prosta analogia z życia

Wyobraź sobie zgadywanie liczby od 1 do 100. Jeśli pytasz „czy to 50?”, a odpowiedź brzmi „za dużo”, od razu odrzucasz liczby 50–100. Następnie pytasz o środek pozostałego zakresu. To dokładnie intuicja wyszukiwania binarnego.

Krótki kontekst historyczny

Idea połowienia zakresu jest starsza niż współczesne języki programowania i pojawia się w klasycznej informatyce jako przykład metody divide and conquer. Binary Search jest jednym z podstawowych algorytmów uczonych przy analizie złożoności, bo bardzo wyraźnie pokazuje różnicę między wzrostem liniowym a logarytmicznym.

Wyjaśnienie krok po kroku

1. Ustal granice

Na początku lewa granica to pierwszy indeks, prawa granica to ostatni indeks.

2. Sprawdź środek

Oblicz indeks środka i porównaj wartość z celem.

3. Odrzuć połowę

Jeśli środek jest za mały, szukaj po prawej. Jeśli za duży, szukaj po lewej.

4. Zakończ

Zwróć indeks po znalezieniu albo informację, że elementu nie ma.

Wizualny walkthrough tekstowy

Szukamy liczby 23 w tablicy:

[2, 5, 8, 12, 16, 23, 38, 56, 72]
  1. Zakres: indeksy 0–8. Środek: indeks 4, wartość 16. Cel 23 jest większy, więc odrzucamy lewą połowę.
  2. Zakres: indeksy 5–8. Środek: indeks 6, wartość 38. Cel 23 jest mniejszy, więc odrzucamy prawą część.
  3. Zakres: indeks 5–5. Środek: indeks 5, wartość 23. Element znaleziony.

Pseudokod

left = 0
right = length(array) - 1
while left <= right:
  middle = floor((left + right) / 2)
  if array[middle] == target:
    return middle
  if array[middle] < target:
    left = middle + 1
  else:
    right = middle - 1
return -1

Przykład w JavaScript

function binarySearch(numbers, target) {
  let left = 0;
  let right = numbers.length - 1;

  while (left <= right) {
    const middle = Math.floor((left + right) / 2);
    const value = numbers[middle];

    if (value === target) {
      return middle;
    }

    if (value < target) {
      left = middle + 1;
    } else {
      right = middle - 1;
    }
  }

  return -1;
}

console.log(binarySearch([2, 5, 8, 12, 16, 23, 38], 23)); // 5

Złożoność O(log n)

Po każdym porównaniu liczba kandydatów spada mniej więcej o połowę. Dla 1 000 000 elementów potrzeba około 20 kroków, a nie miliona. Dlatego czas działania zapisujemy jako O(log n). Pamięć w wersji iteracyjnej wynosi O(1), bo używamy tylko kilku zmiennych pomocniczych.

Najczęstsze błędy

  • Uruchamianie algorytmu na nieposortowanej tablicy.
  • Złe warunki pętli, np. left < right zamiast left <= right w klasycznej wersji.
  • Brak przesunięcia granic o jeden, co może prowadzić do nieskończonej pętli.
  • Mylenie zwracanej wartości: indeks elementu kontra sama wartość.

Kiedy używać, a kiedy nie?

Używaj Binary Search, gdy dane są posortowane i wykonujesz wiele zapytań albo zależy Ci na szybkim wyszukiwaniu. Nie używaj go jako pierwszego wyboru, gdy dane często się zmieniają i sortowanie kosztuje więcej niż samo wyszukiwanie, albo gdy kolekcja jest bardzo mała i prostszy kod liniowy jest wystarczający.

Quiz i praktyka

  1. Dlaczego Binary Search wymaga posortowanych danych?
  2. Ile maksymalnie kroków potrzeba mniej więcej dla 1024 elementów?
  3. Zaimplementuj wersję, która zwraca true/false zamiast indeksu.
  4. Przetestuj algorytm dla wartości mniejszej niż pierwszy element i większej niż ostatni.

Podsumowanie

Binary Search jest fundamentalnym algorytmem, ponieważ uczy myślenia zakresami, redukcji problemu i analizy logarytmicznej. Najważniejszy warunek to posortowane dane. Jeśli jest spełniony, algorytm pozwala bardzo szybko znaleźć element nawet w ogromnej tablicy.
