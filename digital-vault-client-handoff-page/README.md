# KP_Code Digital Vault — Client Handoff Page

## Co to jest?
To samodzielny, premium szablon dokumentu przekazania strony internetowej dla klienta. Działa bezpośrednio w przeglądarce (`index.html`) i jest gotowy do druku / eksportu PDF.

## Dla kogo?
- Freelancerów tworzących strony internetowe
- Małych software house'ów i studiów webowych
- Specjalistów przekazujących klientowi gotowy projekt WWW

## Zawartość
- `index.html` — główny dokument handoff z sekcjami projektowymi i checklistami
- `assets/css/style.css` — styl premium, responsywny i print-friendly (A4)
- `assets/js/main.js` — obsługa checkboxów, resetu i drukowania
- `handoff-template.md` — wersja Markdown do ponownego użycia
- `README.md` — instrukcja użytkowania

## Jak używać
1. Otwórz `index.html` w przeglądarce.
2. Uzupełnij placeholdery danymi projektu i klienta.
3. Przejdź przez checklisty materiałów i weryfikacji.
4. Użyj przycisku **Drukuj / Zapisz PDF** do wygenerowania finalnego dokumentu.

## Jak dostosować
- Edytuj treści sekcji bezpośrednio w `index.html`.
- Zmieniaj branding i tokeny wizualne w `assets/css/style.css` (zmienne CSS w `:root`).
- Rozszerz checklisty dodając nowe `input[type="checkbox"]` z unikalnym `data-check-item`.

## Rekomendowany workflow
1. Uzupełnij dokument po zakończeniu wdrożenia.
2. Zweryfikuj dostępność linków i poprawność danych technicznych.
3. Przejdź z klientem przez checklistę weryfikacyjną.
4. Wygeneruj PDF i poproś o potwierdzenie odbioru.
5. Zachowaj finalną wersję w archiwum projektu.

## Bezpieczeństwo
Nie zapisuj haseł bezpośrednio w dokumencie. Dane dostępowe przekazuj wyłącznie przez bezpieczne kanały (np. menedżer haseł, zaszyfrowane wiadomości).
